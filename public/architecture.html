THE GAME
SESSION
Stateless API routes, self-cancelling debounce, and Supabase Realtime broadcast
app/api/game-session/[id]/
Game Session
API
  • Stateless Next.js API routes on Vercel — no WebSocket server
  • POST /action saves message, sets next_round_at, schedules round via Vercel after()
  • Self-cancelling debounce: rapid actions extend the timer; only the last worker fires
  • Optimistic lock: round_in_progress + 409 guard drops overlapping rounds
  • Claude tokens streamed to all clients via Supabase Realtime broadcast

Architecture
Key Design Decisions
01
Next.js API Routes + Vercel after()
Game session migrated from Deno WebSocket to stateless Next.js API routes. Supabase Edge wall-clock limits caused 1006 disconnects mid-stream. Vercel after() schedules deferred round workers without holding a connection open.
02
Optimistic Race Guard
round_in_progress flag prevents duplicate opening narrations when multiple players connect simultaneously to a fresh campaign.
03
Messages Table as Memory
The full game history in the messages table IS the AI's memory. Rebuilt into Anthropic conversation format on every round — no separate memory store needed.
04
Debounced Action Batching
Player messages queued and debounced before calling Claude. Multiple players typing simultaneously produce a single coherent narration round, not fragmented responses.
05
Prompt Caching
System prompt and last history message cache-anchored on every Claude call. With long campaigns, this significantly reduces token cost and narration latency.
06
Magic Link Auth via Supabase
Frictionless onboarding — no passwords. Session cookie handled by Next.js server components via Supabase SSR helpers. No special transport needed.
